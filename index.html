📖 Issue Type Reference

Use these labels when logging UX bugs in Jira, Linear, or GitHub Issues.

🟠 Error handling — missing loading, error, success states
🔴 Empty states — blank screens with no guidance
🟢 Navigation — unclear structure, findability
🔵 Onboarding — first-run, setup flow, time-to-value
🟣 Accessibility — contrast, keyboard nav, VoiceOver
🟡 Copy — CTAs, labels, error messages, tone
🩷 Visual design — spacing, color, consistency
🌸 Cognitive overload — too much at once, decision fatigue
🌺 Functional bugs — broken interactions, missing feedback
📊 Scoring Guide
9–10Exceptional — sets the standard for the category
7–8Good — works well, a few improvements available
5–6Average — notable friction affects experience
3–4Poor — significant problems affect core tasks
1–2Broken — core flows fail or confuse users
